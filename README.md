# Minute Factory: AI 회의록 자동 정리 및 Q&A 프로젝트

이 프로젝트는 음성 회의록을 자동으로 텍스트로 변환 및 요약하고, 저장된 회의록을 기반으로 질문에 답변하는 AI 기반 웹 애플리케이션입니다.

여러 팀원(유수현, 성정현, 손용훈)이 각자 개발한 입력, 처리 모듈을 하나로 통합하고, Gemini-CLI를 통해 CRAG 기반의 Q&A 챗봇 기능을 추가로 통합 및 고도화한 버전입니다.

## 주요 기능

- **웹 기반 UI**: Gradio를 사용하여 누구나 쉽게 사용할 수 있는 웹 인터페이스를 제공합니다.
- **다양한 입력 방식 지원**:
    - **파일 업로드**: `mp3`, `m4a`, `mp4` 등 다양한 오디오/비디오 파일을 업로드하면 자동으로 `wav` 음성 파일로 변환됩니다.
    - **실시간 녹음**: 웹페이지에서 직접 마이크를 사용해 음성을 녹음하고 저장할 수 있습니다.
- **AI 처리 파이프라인**:
    - **화자 분리**: `pyannote.audio`를 사용해 대화에 참여한 여러 명의 화자를 식별합니다.
    - **음성 인식(STT)**: `OpenAI Whisper`를 사용해 음성을 텍스트로 정확하게 변환합니다.
    - **자동 교정 및 요약**: 처리된 텍스트를 LLM을 통해 문맥에 맞게 교정하고, 전체 회의 내용을 핵심만 요약합니다.
- **LLM 모델 선택**: 교정 및 요약 작업에 사용할 AI 모델을 `GPT-4`와 `Gemini-Pro` 중에서 선택할 수 있습니다.
- **결과 저장 및 벡터화**: 요약, 교정본 등 모든 결과물을 파일로 저장하고, 이 중 교정된 텍스트는 `ChromaDB` 벡터 저장소에 자동으로 저장됩니다.
- **회의록 검색 Q&A (CRAG)**:
    - 저장된 회의록을 선택하여 내용에 대해 자유롭게 질문하고 답변을 받는 챗봇 기능입니다.
    - 회의록에 답변이 없는 경우, `Tavily` 웹 검색을 통해 인터넷에서 정보를 찾아 답변하는 Corrective RAG(CRAG) 기술이 적용되어 있습니다.

## 프로젝트 구조

```
minute_code/

├── ui/                     # UI 관련 모듈
│   ├── app_layout.py       # 전체 UI 레이아웃 및 콜백
│   └── input_handlers.py   # 오디오 입력(업로드, 녹음) 처리
├── core_processing/      # 핵심 기능 처리 모듈
│   ├── main_pipeline.py    # 오디오 처리 전체 파이프라인
│   ├── stt_diarization.py  # STT 및 화자 분리
│   └── llm_tasks.py        # LLM 기반 교정 및 요약
├── chatbot/                # 챗봇 관련 모듈
│   └── crag_logic.py       # CRAG 챗봇 로직
└── utils/                  # 공통 유틸리티 모듈
    ├── api_keys.py         # API 키 로드
    ├── config.py           # 전역 설정
    └── file_manager.py     # 파일 저장/관리
├── chroma_db/            # (자동생성) ChromaDB 벡터 저장소
├── data/                 # (자동생성) 입력 오디오 파일 저장
├── results/              # (자동생성) 결과물 저장
├── temp/                 # (자동생성) 임시 파일 저장
├── .env                  # (직접생성) API 키 저장 파일
└── requirements.txt      # 필요 라이브러리 목록
```

## 설치 및 실행 방법

1.  **프로젝트 폴더로 이동**

2.  **(권장) 가상 환경 생성 및 활성화**:
    ```bash
    python -m venv venv
    .\venv\Scripts\activate
    ```

3.  **필요 라이브러리 설치**:
    ```bash
    pip install -r requirements.txt --upgrade
    ```
    *   **참고**: `torch`나 `ffmpeg` 설치에 문제가 발생하면 `requirements.txt` 파일 안의 주석을 참고하여 수동으로 설치해주세요.

4.  **API 키 설정**:
    *   프로젝트 루트 폴더에 `.env` 라는 이름의 파일을 만드세요.
    *   파일을 열고 아래와 같이 필요한 API 키를 모두 입력하세요. **(TAVILY_API_KEY 추가됨)**
        ```
        OPENAI_API_KEY="sk-..."
        GOOGLE_API_KEY="AIza..."
        PYANNOTE_TOKEN="hf_..."
        TAVILY_API_KEY="tvly-..."
        ```

5.  **애플리케이션 실행**:
    ```bash
    python minute_code/main_app.py
    ```6.  **웹 브라우저에서 접속**:
    *   터미널에 표시되는 주소(보통 `http://127.0.0.1:7860`)를 웹 브라우저에 열어 애플리케이션을 사용합니다.

## 사용 방법

### 1. 회의록 생성

1.  **음성 파일 준비**: **'1. 음성 파일 관리'** 탭 또는 **'2. 음성 녹음'** 탭을 사용하여 처리할 음성 파일을 `data` 폴더에 준비합니다.
2.  **처리 및 요약 실행**: **'4. 처리 & 요약'** 탭으로 이동하여 원하는 파일을 선택하고 **'처리 시작'** 버튼을 클릭합니다.
3.  **결과 확인**: 작업이 완료되면 요약 및 교정된 대화록을 확인합니다. 결과 파일은 `results` 폴더에, 검색 가능한 인덱스는 `chroma_db` 폴더에 저장됩니다.

### 2. 회의록 검색 Q&A

1.  **Q&A 탭으로 이동**: **'5. 회의록 검색 Q&A'** 탭으로 이동합니다.
2.  **회의록 선택**: 드롭다운 메뉴에서 질문하고 싶은 회의록을 선택합니다.
3.  **질문하기**: 하단의 입력창에 질문을 입력하고 '전송' 버튼을 누르면 챗봇의 답변을 확인할 수 있습니다.
