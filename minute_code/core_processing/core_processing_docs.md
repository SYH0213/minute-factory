# 핵심 처리 모듈 (`core_processing/`)

`core_processing` 폴더는 음성 회의록을 텍스트로 변환하고, 교정하며, 요약하는 핵심 비즈니스 로직을 포함합니다.

## 주요 역할
- **전체 처리 파이프라인 관리**: 오디오 입력부터 최종 요약 및 저장까지의 전체 과정을 조율하고 실행합니다.
- **음성 인식 (STT) 및 화자 분리**: 오디오 파일에서 텍스트를 추출하고, 대화 참여자를 식별합니다.
- **LLM 기반 텍스트 처리**: 추출된 텍스트를 LLM(Large Language Model)을 사용하여 문맥에 맞게 교정하고, 회의 내용을 요약합니다.

## 주요 파일
- `main_pipeline.py`: 전체 오디오 처리 파이프라인의 진입점입니다. 화자 분리, STT, LLM 교정 및 요약, 결과 저장, 벡터 저장소 업데이트 등의 단계를 순차적으로 실행합니다.
- `stt_diarization.py`: `pyannote.audio`를 이용한 화자 분리(Diarization)와 `OpenAI Whisper` API를 이용한 음성 인식(STT) 기능을 제공합니다.
- `llm_tasks.py`: `GPT-4o` 또는 `Gemini-2.5-pro`와 같은 LLM을 사용하여 텍스트 교정 및 회의 요약을 수행하는 함수들을 포함합니다.
