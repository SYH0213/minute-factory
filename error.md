왜 영어/한자가 나올까?

자동 언어 감지 오차

Whisper는 입력 구간별로 “이게 한국어인지, 영어인지, 중국어인지”를 추정해.

짧은 구간(특히 1초 미만, 단어 하나 수준)은 음향적 단서가 적어서 잘못 감지 → 엉뚱하게 영어/한자로 추출되기도 함.

노이즈/잡음/기침/웃음

잡음, 호흡음 같은 걸 Whisper가 “문자화”하려고 하면서 엉뚱한 외국어 단편으로 출력.

모델 특성

Whisper는 멀티링궐(multi-lingual)이라서, 애매한 발음을 강제로 어떤 언어 단어에 매핑해 버림. → 예: “음”을 um 으로, “허”를 중국어 간체 한 글자처럼.

해결 방법
1. 언어 강제 설정

Whisper API 호출 시 language="ko" 옵션을 주면 언어 자동 감지가 아니라 한국어 고정으로 동작해.

이러면 영어/한자 섞임 현상이 상당히 줄어들어.

result = model.transcribe("audio.mp3", language="ko")

2. 짧은 구간 필터링

1초 미만 구간은 대부분 의미 없는 잡음일 가능성이 높음 → 버려도 무방.

다만, “네”, “네네”, “아” 같은 짧은 맞장구는 실제 회의 문맥에서 중요할 때가 있음.

따라서 완전 삭제보다는 조건부 삭제가 좋아:

(a) 1초 미만 AND 글자 수 2 이하 → 삭제

(b) 1초 미만인데도 “네, 아, 예, 음” 같은 사전에 등록된 짧은 응답이면 유지

3. 후처리 필터

정규식이나 간단한 rule-based 후처리로 한자/영어 단어를 걸러내는 방식.
예:

import re

def clean_text(text):
    # 한자/영어만 있고 길이가 1~2자인 경우 제거
    if re.fullmatch(r"[A-Za-z\u4e00-\u9fff]{1,2}", text.strip()):
        return ""
    return text

4. 후속 LLM 교정에 맡기기

Whisper 결과 전체를 LLM에 넘길 때:
“회의는 한국어로만 진행되었으니, 영어/중국어 같은 잘못된 출력은 모두 제거하고 문맥에 맞게 교정해 주세요”
라는 프롬프트를 추가 → 자동 보정.

결론

Whisper에서 language="ko" 강제 설정 + 짧은 구간 필터링을 1차 적용.

그 후 LLM 교정 파이프라인에서 잡아내면 가장 안정적.

단순히 “1초 미만은 다 없애기”보다는, 짧아도 회의 흐름상 중요한 응답(“네”, “아”)은 유지하는 쪽이 좋아.
