# Gemini-CLI 작업 기록

## ❗ 중요 작업 원칙

- **절대 내가 허락하기 전에 파일을 수정하거나 코드를 수정하지 않는다.**

---

## 🎯 프로젝트 목표

유수현, 성정현, 손용훈, 심세아 4인의 파이썬 코드를 병합하여, 음성 파일을 입력받아 STT, 화자 분리, LLM 기반 교정 및 요약을 수행하는 단일 웹 애플리케이션(`Minute Factory`)을 개발. 최종적으로 회의록 기반의 Q&A가 가능한 챗봇 기능까지 통합한다.

---

## ✅ 초기 완료된 작업 (통합 빌드)

1.  **코드 분석**: 각 개발자의 코드 분석 완료.
2.  **병합 계획 수립**: 손용훈님 코드를 Base로 하여 병합 계획 수립.
3.  **`minute_factory` 프로젝트 생성**: 기본 프로젝트 구조 및 파일 생성.
4.  **`processing` 백엔드 모듈 구현**: 각 기능 모듈화 및 통합.
5.  **`main_app.py` 프론트엔드 UI 구현**: UI와 백엔드 연동.

## 📝 Gemini-CLI 업데이트 기록 (2025-09-29)

### CRAG 챗봇 기능 통합 및 전체 리팩토링

- **오디오 입력부 모듈화**
    - `main_app.py`에 있던 파일 업로드, 음성 녹음 관련 UI 및 비즈니스 로직을 `processing/audio_input.py` 모듈로 분리하여 코드의 가독성과 유지보수성을 향상.

- **CRAG 기반 Q&A 챗봇 기능 신규 통합**
    - `crag_meeting_minutes.py` 코드를 기반으로, `ChromaDB`를 벡터 저장소로 사용하는 `processing/chatbot_crag.py` 모듈 신설.
    - 회의록 처리 완료 시, 교정된 텍스트(`corrected_...txt`)가 `ChromaDB`에 자동으로 저장 및 업데이트되도록 `pipeline.py` 수정.
    - `main_app.py`에 "회의록 검색 Q&A" 탭을 추가하고, 사용자가 과거 회의록을 선택하여 대화할 수 있는 UI 구현.
    - `Tavily` 웹 검색을 연동하여, 회의록에 없는 내용은 인터넷에서 검색하여 답변하는 CRAG(Corrective RAG) 기능 완전 구현.

- **오류 수정 및 안정성 강화**
    - **파일 업로드 오류 해결**: 모듈 분리 과정에서 발생한 `NoneType` 오류를 `upload_wrapper` 함수를 통해 해결. Gradio의 프로그레스 바 기능이 정상적으로 동작하도록 수정.
    - **챗봇 기능 오류 해결**: 웹 검색 결과 처리 시 발생하던 `TypeError`를 해결하여 챗봇 기능의 안정성 확보.
    - **라이브러리 경고 해결**: 최신 버전에 맞게 `Tavily`, `Chroma` 등 LangChain 관련 라이브러리의 `import` 구문과 사용법을 수정하여 모든 Deprecation 경고를 제거.
    - **UI/UX 개선**: Gradio 챗봇의 UI 포맷을 최신 `messages` 타입으로 업데이트하고, 관련 콜백 함수 로직을 수정.

### 프로젝트 구조 리팩토링 (협업 효율성 증대)

- **목표**: 팀원(유수현:UI, 성정현/손용훈:핵심처리, 심세아:챗봇)이 각자 담당 코드에 집중할 수 있도록 프로젝트 구조를 역할 기반으로 재구성.
- **주요 변경 사항**:
    - `processing` 폴더를 `ui`, `core_processing`, `chatbot`, `utils`의 4개 역할별 폴더로 분리.
    - `main_app.py`에서 UI 생성 로직을 `ui/app_layout.py`로 이전하여, `main_app.py`는 앱 실행 역할만 담당하도록 간소화.
    - 파일 이동 및 이름 변경에 따라 프로젝트 전체의 `import` 구문을 수정하여 새로운 구조에 맞게 업데이트.
    - **최상위 패키지 구조 도입**: 모든 파이썬 소스 코드(main_app.py, ui/, core_processing/, chatbot/, utils/)를 `minute_factory/`라는 단일 최상위 패키지 폴더 안으로 이동하여 프로젝트 구조의 명확성과 관리 용이성을 극대화. `README.md` 및 실행 명령어 업데이트 완료.
    - **최상위 패키지 폴더명 변경**: `minute_factory` 폴더명을 `minute_code`로 변경하여 프로젝트 이름과의 혼동을 방지하고 코드의 의미를 명확히 함. `README.md` 및 실행 명령어 업데이트 완료.
